# Audit Code — Audit Comparé (audit-compare)
Date : 2026-07-08 · Auditeur : Senior Crawlers Auditor · Périmètre : Backend (Edge), Frontend (React), Tables.

## Synthèse Executive
L'audit révèle une fonctionnalité "Battle Card" très puissante mais souffrant d'une architecture monolithique critique (1062 l. backend, 1677 l. frontend) et de fuites de coûts LLM. Le contournement de l'AI Gateway et l'absence de rate-limit IP pour les appels payants (DataForSEO) sont les priorités P0.

## Findings P0/P1 (Critiques & Prioritaires)

### #149 · P0 · 2j — Fuite de coûts LLM & AI Gateway bypass
Le backend utilise `fetch('https://openrouter.ai/...')` en direct au lieu de `aiGatewayFetch` (ou `openRouterAI`).
- **Risque** : Pas de rotation de clé, pas de fallback, pas de monitoring centralisé des coûts, bypass des quotas.
- **Fix** : Migrer `generateSeedsWithAI`, `analyzeSite` et `runCrossComparison` vers `aiGatewayFetch`.

### #150 · P1 · 1j — Absence de Rate-Limit IP (Lead Magnet Drain)
Les appels payants DataForSEO (Backlinks + SERP + KW) sont déclenchables sans restriction IP.
- **Risque** : Un script peut vider le compte DataForSEO en quelques minutes via le endpoint anonyme.
- **Fix** : Implémenter `ipRateLimiter.ts` (shared) sur le handler principal avant les appels payants.

### #151 · P1 · 1.5j — Monolithe Frontend & Re-renders massifs
`AuditCompare.tsx` (1677 lignes) contient toute la logique, les types, le i18n et 8 composants complexes non mémoïsés.
- **Risque** : Performance dégradée (re-renders du RadarChart à chaque step), maintenabilité quasi-nulle.
- **Fix** : Splitter en `features/audit-compare/components/*` (`CompareForm`, `RadarBattle`, `SerpBattlefield`, etc.).

### #152 · P1 · 1j — SSRF & Timeout Lack sur extractPageMetadata
La fonction `extractPageMetadata` fetch n'importe quelle URL utilisateur sans validation stricte.
- **Risque** : SSRF (Server-Side Request Forgery) sur le réseau interne Supabase.
- **Fix** : Utiliser `assertSafeUrl` de `_shared/ssrf.ts` et `stealthFetch`.

## Findings P2 (Optimisations)

### #153 · P2 · 0.5j — TTL Cache inadapté (Hardcoded 2h)
Le cache est forcé à 2h (`saveToCache`) alors que les backlinks et keywords changent mensuellement.
- **Risque** : Appels payants redondants pour les mêmes domaines dans la même journée.
- **Fix** : Passer à 24h minimum, idéalement 7j pour les données "froides" (backlinks).

### #154 · P2 · 1j — Monolithe Backend (1062 l.)
Le fichier `index.ts` contient 17 fonctions internes (extraction HTML, couleurs, parsing JSON).
- **Risque** : Duplication de logique (ex: extraction de titre déjà présente ailleurs).
- **Fix** : Déporter la logique dans `_shared/audit-compare/*` ou utiliser les utilitaires partagés (`pageMetadata.ts`).

### #155 · P2 · 0.5j — SPA Fallback inefficace
Le fallback Browserless est déclenché sur un simple check de longueur de texte (`textOnly.length < 200`).
- **Risque** : Faux positifs déclenchant des appels Browserless coûteux inutilement.
- **Fix** : Utiliser `isSpaDetected` (shared) basé sur les signatures de frameworks (Next, Vite, etc.).
