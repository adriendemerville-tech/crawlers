# Audit Brief — Audit Comparé (audit-compare)
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération

## 1. Objectif métier
Comparer **2 URLs** (site utilisateur + concurrent, ou 2 concurrents) sur les axes SEO/GEO/Content/Backlinks/PageSpeed pour identifier les écarts stratégiques et les leviers actionnables. Feature "battle card" positionnée comme démonstrateur produit vs Semrush/SimilarWeb.

## 2. Cas d'usage principaux
- **Anonyme / Free** → colle 2 URLs sur `/audit-compare` → reçoit dashboard côte-à-côte (mots-clés, contenu, backlinks, vitesse, cross-comparaison SERP) → email gate.
- **Premium** → comparaisons illimitées + cache 24h + analyse cross-SERP profonde + rapport téléchargeable.
- **Pro Agency** → comparaisons multi-clients, snapshot historique.
- **Admin** → voit toutes les comparaisons, débogue le cache.

## 3. Logique métier clé
- **Pipeline** (`audit-compare/index.ts` 1 062 l.) :
  1. `extractPageMetadata` (fetch HTML + parse titre/desc/H1/depth/couleur brand)
  2. `fetchPageSpeedScores` (Google PageSpeed API)
  3. `fetchBacklinkProfile` (DataForSEO backlinks/summary)
  4. `generateSeedsWithAI` (LLM produit N mots-clés seed à partir du contexte)
  5. `fetchKeywordData` (DataForSEO keyword_data pour ranks + volumes)
  6. `analyzeSite` (LLM synthétise fiche site)
  7. `runCrossComparison` (LLM compare les 2 fiches sur axes prédéfinis)
  8. `crossCheckSerpRankings` (DataForSEO SERP live pour vérifier les positions déclarées)
  9. `saveToCache` (`audit_cache`)
- **Dépendances externes** : DataForSEO (SERP + backlinks + keyword_data) — **jusqu'à 6 appels payants par comparaison**, Google PageSpeed API, LLM (à identifier — direct fetch ou aiGatewayFetch ?)
- **Tables** : `audit_cache` (6 col, 1 pol), pas de persistence longue-durée par défaut (les résultats vivent dans le cache 24h)
- **Contraintes projet** : mem://tech/ai/llm-task-alignment-fr (routing LLM), mem://tech/architecture/keyword-universe-ssot-fr (SSOT KW — à respecter, ne pas dupliquer les seeds côté audit-compare), mem://tech/security/multi-tenant-isolation-fr

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |
|---|---|---|
| Fonctionnel | 5 axes retournés sans crash sur 3 paires URLs types | test manuel |
| Performance | latence p95 < 60s (6 API + 3 LLM) | logs edge |
| Coût | < 15¢ / comparaison (DataForSEO x6 + LLM x3) | `paid_api_calls` + `ai_gateway_usage` |
| Cache | hit ratio > 30% sur URLs répétées | `audit_cache` count vs invocations |
| Sécurité | anti-drain anonyme (rate limit IP), SSRF check sur les 2 URLs | `security--run_security_scan` |
| Modularité | Aucun fichier > 500 lignes | wc -l |

## 5. Benchmark concurrentiel
- **Semrush Domain vs Domain** : mature, données massives, mais interface figée et coût ~130€/mois. Force : profondeur data. Faiblesse : pas de LLM cross-analysis, pas de reco actionnable.
- **SimilarWeb Compare** : trafic estimé + audience, faible côté SEO on-page. Force : traffic intelligence. Faiblesse : pas de mots-clés fins, pas de PageSpeed.
- **Ahrefs Site Comparison** : backlinks + KW rankings solides. Force : profil backlinks. Faiblesse : pas de LLM synthèse, pas de PageSpeed, pas de GEO.
- **SEObility / Sistrix** : équivalent Semrush allemand/européen. Faiblesse identique : pas de LLM.
- **Barre à dépasser** : Crawlers combine **DataForSEO backlinks + PageSpeed + KW rankings + LLM cross-analysis + SERP re-vérifié** en un seul rapport free/anonyme. **La cross-analyse LLM (`buildCrossComparePrompt`) est le vrai différenciateur** — aucun concurrent grand public ne fait ça en un click.

## Hypothèses à valider pendant l'audit
1. **Monolithe 1 062 l.** + **17 fonctions internes** → découpe naturelle par étape pipeline (`_shared/auditCompare/{metadata,pagespeed,backlinks,seeds,keywords,siteAnalysis,crossCompare,serpCheck,cache}.ts`)
2. **LLM hors gateway** (rappel Vague 1 : `audit-compare` était dans les 3 offenders) → vérifier `generateSeedsWithAI`, `analyzeSite`, `runCrossComparison` — utilisent-ils `aiGatewayFetch` ou direct `fetch('openrouter'/'openai')` ?
3. **Cache trop court** : TTL 24h alors que les backlinks/keywords ne bougent quasi-jamais en 24h. Augmenter TTL différencié par type ? (PageSpeed 6h, KW 7j, backlinks 30j)
4. **Aucun `paid_api_calls` visible** : `trackPaidApiCall` est importé mais chaque appel externe est-il tracé ? DataForSEO x3 (SERP + backlinks + KW) + PageSpeed x2 = 5 appels/comparaison à tracer.
5. **`AuditCompare.tsx` 1 677 l.** : le plus gros front du projet. À splitter en `<CompareForm>`, `<SiteMetadataCard>`, `<KeywordComparisonTable>`, `<BacklinkComparisonChart>`, `<CrossAnalysisPanel>`, `<PageSpeedGauge>`.
6. **Anti-drain anonyme** : `checkIpRate` / `acquireConcurrency` importés ? Si non, chaque anonyme peut déclencher **6 appels payants** gratuitement.
7. **SSRF** : `assertSafeUrl` importé pour les 2 URLs ? Sinon vecteur d'attaque interne.
8. **Doublon `audit-competitor-url`** (334 l., feature Pro Agency Concurrence) — même besoin fonctionnel ? Fusion possible en un moteur paramétré (mode=`public_lead` vs `tracked_competitor`) ?
