## 1. Résumé exécutif

Le moteur sémantique Cocoon est **fonctionnellement complet** (11 edges, 3 modes graphe, chat stratégiste, Content Architect, cron refresh) et constitue un avantage concurrentiel réel face aux concurrents listés au brief. La base RLS est globalement saine — ENABLE ROW LEVEL SECURITY sur l'ensemble des 11 tables, isolation `user_id = auth.uid()`.

**Trois zones de risque** émergent à l'audit :
1. **P0 RLS** : `cocoon_strategy_plans` et `cocoon_diagnostic_results` n'ont **pas de policy INSERT** pour `authenticated` — les edges y écrivent via `service_role` mais aucun front ne peut créer ces entités directement.
2. **P0 Prompt injection** : `domainDataBlock` (contenu brut crawlé du site tiers) est concaténé sans sanitisation dans le system prompt de `cocoon-chat` — vecteur d'injection depuis les pages du domaine analysé.
3. **P1 Bundle bloat 3D** : `three.js` + `@react-three/fiber` + `@react-three/drei` importés **statiquement** dans `CocoonForceGraph3D.tsx` (~520 KB gzip), chargés pour tous les utilisateurs de `/cocoon` y compris ceux restant en mode 2D/radial.

Violations design system : 23 emojis dans `CocoonAIChat.tsx` et dans les prompts système (`🎯 📝 💻 ⚙️ 🔴 🟡 🟢 👋 💡`), 1 occurrence `blue-500` — contraires aux règles Crawlers (pas d'emoji, pas de bleu IA).

## 2. Findings

### C1 · P0 · `cocoon_strategy_plans` : policy INSERT manquante
**Preuve** : `migrations/20260321194146_*.sql` l.48-57 — seuls `FOR SELECT` et `FOR UPDATE` sont créés pour `authenticated`. Aucun `FOR INSERT`.
**Impact** : Si un front tente d'insérer un plan directement (RPC, Supabase client), retour `permission denied`. Seul `service_role` peut écrire → tout front sans edge intermédiaire est bloqué.
**Fix** :
```sql
CREATE POLICY "Users can insert own strategy plans"
  ON public.cocoon_strategy_plans FOR INSERT TO authenticated
  WITH CHECK (user_id = auth.uid());
```
**Effort** : 0,25j · **Score** : 3/10

---

### C2 · P0 · `cocoon_diagnostic_results` : policy INSERT manquante
**Preuve** : même migration — seul `FOR SELECT` + service_role `FOR ALL`. Les 5 edges `cocoon-diag-*` écrivent via `service_role` (OK), mais le frontend ne peut pas insérer sans passer par une edge.
**Impact** : identique à C1. Cohérence à maintenir si un futur RPC expose l'upsert.
**Fix** :
```sql
CREATE POLICY "Users can insert own diagnostics"
  ON public.cocoon_diagnostic_results FOR INSERT TO authenticated
  WITH CHECK (user_id = auth.uid());
```
**Effort** : 0,25j · **Score** : 4/10

---

### C3 · P0 · `cocoon-chat` : `domainDataBlock` injecté sans `<user_input>` wrapping
**Preuve** : `cocoon-chat/index.ts` l.60-100 — `getDomainContext()` retourne des blocs construits depuis le contenu crawlé du domaine tiers (titres, métas, contenus de pages). Ces blocs sont concaténés directement dans `basePrompt` (l.490-495) sans passer par `wrapUserInput()` ni aucune sanitisation.
**Impact** : un opérateur de site malveillant peut injecter `Ignore all previous instructions` dans le contenu de ses pages, remonté par le crawler, et manipuler le LLM en mode chat stratégiste. Pattern identique au finding M10 (Marina audit).
**Fix** : importer `wrapUserInput` de `_shared/promptSafety.ts` et wrapper `ctx.blocks` avant concaténation :
```ts
import { wrapUserInput } from '../_shared/promptSafety.ts';
// ...
domainDataBlock = `\n\nDONNÉES DOMAINE :\n${wrapUserInput(ctx.blocks.join('\n\n'))}`;
```
**Effort** : 0,25j · **Score** : 3/10

---

### C4 · P1 · Bundle 3D non lazy-loadé (~520 KB gzip bloquants)
**Preuve** : `CocoonForceGraph3D.tsx` l.2-4 — `import { Canvas, useFrame, extend } from "@react-three/fiber"`, `import { OrbitControls, Html, Billboard, Text, Line } from "@react-three/drei"`, `import * as THREE from "three"`. `src/pages/Cocoon.tsx` importe `CocoonForceGraph3D` statiquement l.11, sans `React.lazy`/`Suspense`.
**Impact** : Three.js (~600 KB raw, ~180 KB gzip) + Drei (~200 KB) téléchargés au premier rendu de `/cocoon` même pour les utilisateurs en mode 2D ou radial. FPS < 30 probable sur mobile/low-end.
**Fix** :
```tsx
// Cocoon.tsx
const CocoonForceGraph3D = React.lazy(() =>
  import('@/components/Cocoon/CocoonForceGraph3D').then(m => ({ default: m.CocoonForceGraph3D }))
);
// Wrapper : <Suspense fallback={<GraphSkeleton />}>
```
**Effort** : 0,5j · **Score** : 5/10

### C5 · P1 · `CocoonAIChat.tsx` = 2 143 lignes — God component
**Preuve** : `wc -l src/components/Cocoon/CocoonAIChat.tsx` → 2143. Contient : gestion session, fetch site context, autonomy, domain data, polling, quick replies, quiz trigger, markdown rendering, toolbar, export, mode stratégiste vs normal.
**Impact** : impossible à tester unitairement, re-renders coûteux (19 `: any` dans `Cocoon.tsx` suggèrent des props non typées), coupling fort. Premier vecteur de régression.
**Fix** : extraire au minimum `useCocoonChat` hook (state + fetch, ~600 l.) + `CocoonChatMessages` (rendu, ~400 l.) + `CocoonChatInput` (input + quick replies, ~300 l.) + `CocoonChatSession` (session persist, ~200 l.).
**Effort** : 1,5j · **Score** : 5/10

---

### C6 · P1 · Emojis dans prompts système et UI — violation design system
**Preuve** :
- `cocoon-chat/index.ts` l.370-410 : `🔴 🟡 🟢 📝 💻 ⚙️ 🎯` dans le system prompt stratégiste
- `CocoonAIChat.tsx` l.451-531 : `👋` dans les messages de greeting (3 occurrences)
- `CocoonAIChat.tsx` l.928 : `💡` dans un message push quiz
- Brief §4 critère UX : *"pas d'emoji"*
**Impact** : incohérence avec le design system Crawlers, régression possible si le brief est appliqué au runtime (filtrage emojis = messages brisés).
**Fix** : remplacer les emojis colorés dans le prompt par des préfixes texte (`[CRITIQUE]`, `[ATTENTION]`, `[INFO]`) ; retirer les 👋 / 💡 des greeting strings.
**Effort** : 0,5j · **Score** : 6/10

---

### C7 · P1 · `cocoon_strategy_plans` : policy DELETE absente
**Preuve** : `migrations/20260321194146_*.sql` — aucun `FOR DELETE` pour `authenticated`. Un utilisateur ne peut pas supprimer ses propres plans depuis le front.
**Impact** : dette UX (pas de "reset plan"), contournement uniquement via edge admin.
**Fix** :
```sql
CREATE POLICY "Users can delete own strategy plans"
  ON public.cocoon_strategy_plans FOR DELETE TO authenticated
  USING (user_id = auth.uid());
```
**Effort** : 0,25j · **Score** : 6/10

---

### C8 · P1 · `cron-cocoon-refresh` : limite 200 sites non paginée — risque timeout
**Preuve** : `cron-cocoon-refresh/index.ts` l.33 — `.limit(200)` fixe. Avec 200 sites × 1,5 s/site = 300 s, budget MAX_RUNTIME_MS = 480 s. À partir de ~200 sites actifs le cron sera systématiquement tronqué sans reprise d'état (`last_processed_id`).
**Impact** : sites en queue jamais rafraîchis si croissance > 200 abonnés Premium+. Aucun monitoring d'alerte sur `skipped` (la variable existe mais reste toujours à 0).
**Fix** : remplacer `limit(200)` par pagination avec `order('last_cocoon_refresh_at', nullsFirst)` + incrémenter `skipped` correctement + envoyer alerte Slack si `sitesList.length >= 200`.
**Effort** : 0,5j · **Score** : 6/10

---

### C9 · P2 · `CocoonAIChatUnified.tsx` = ré-export inutile (9 lignes)
**Preuve** : `src/components/Cocoon/CocoonAIChatUnified.tsx` l.1-9 — fichier entier = commentaire + `export { CocoonAIChat as CocoonAIChatUnified }`.
**Impact** : indirection trompeuse, confusion pour les futurs développeurs, import circulaire potentiel.
**Fix** : chercher/remplacer `CocoonAIChatUnified` par `CocoonAIChat` dans tous les consumers, supprimer le fichier.
**Effort** : 0,25j · **Score** : 7/10

---

### C10 · P2 · `cocoon-chat` : pas de cap tokens sur `domainDataBlock`
**Preuve** : `cocoon-chat/index.ts` l.586 — `totalPromptChars` est *loggué* mais pas *limité*. `getDomainContext` peut retourner des dizaines de blocs (crawl complet). Résultat : prompts de 80 000+ tokens possibles sur grands sites → coût > 2¢/message (cible brief §4).
**Impact** : dépassement budget LLM, latence 10-30 s, erreurs `context_length_exceeded`.
**Fix** : tronquer `domainDataBlock` à MAX 4 000 chars avant injection :
```ts
if (domainDataBlock.length > 4000) domainDataBlock = domainDataBlock.slice(0, 4000) + '\n[...tronqué]';
```
**Effort** : 0,25j · **Score** : 7/10
