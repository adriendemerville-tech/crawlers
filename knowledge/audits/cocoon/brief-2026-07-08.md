# Audit Brief — Cocoon
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération

## 1. Objectif métier
Visualiser et piloter le maillage interne d'un site (semantic graph 2D/3D/radial) + chat IA stratégique dédié + Content Architect pour créer/optimiser les pages du cocon (structure, tonalité, images, quick wins).

## 2. Cas d'usage principaux
- **Premium+** → sélectionne un site → visualise le cocon 3D → identifie clusters faibles / cannibalisation
- **Pro Agency** → utilise Bulk Auto-Linking pour appliquer maillage sur tout le site
- **Tout plan** (avec quota) → chat CocoonAIChat = dialogue avec le Stratège sur les nodes sélectionnés
- **Content Architect** → génère un draft complet (structure H1-H3, tonalité, images, JSON-LD, tasks)
- **Cron hebdo** → `cron-cocoon-refresh` recalcule les liens auto

## 3. Logique métier clé
- **Comprehensive engine v3** : cluster detection, cannibalisation, saturation (mem `cocoon/comprehensive-engine-v3-fr`)
- **Automated task plan** : linking async polling (mem `cocoon/automated-task-plan-fr`)
- **Visual controls v4** : X-Ray mode, zoom, filters (mem `cocoon/visual-controls-v4-fr`)
- **Depth logic** : BFS depth ↔ strategy impact (mem `cocoon/depth-logic-fr`)
- **Page intent** : Know/Do/Buy/Navigate/Unknown détecté au crawl (mem `page-intent-and-mesh-audit-fr`)
- **Content Architect markdown cache** : TTL 30j + RPC `get_shared_architect_recommendation` (mem `content-architect/markdown-cache-fr`)
- **Backend** : `cocoon-strategist` (déterministe), `calculate-cocoon-logic`, `cocoon-chat` (LLM), `cocoon-diag-{authority,content,semantic,structure,subdomains}`, `cocoon-{auto,bulk-auto,deploy}-linking`, `cocoon-batch-deploy`, `persist-cocoon-session`, `cron-cocoon-refresh`
- **Tables** : `cocoon_sessions`, `cocoon_tasks`, `cocoon_recommendations`, `cocoon_chat_histories`, `cocoon_auto_links`, `cocoon_diagnostic_results`, `cocoon_batch_operations`, `cocoon_theme_settings`, `cocoon_strategy_plans`, `cocoon_architect_drafts`, `content_architect_cache`

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |
|---|---|---|
| Fonctionnel | 3 graphes rendent >100 nodes sans lag | manuel + FPS |
| Performance | Rendu 3D 60 FPS jusqu'à 500 nodes | React Profiler |
| Coût LLM | Chat < 2¢/message | `ai_gateway_usage` |
| Sécurité | RLS par `user_id` sur tables cocoon | scan |
| UX | Design system Crawlers (violet/or/noir/blanc), pas d'emoji, pas de bleu IA | revue visuelle |

## 5. Benchmark concurrentiel
| Concurrent | Force | Faiblesse vs Cocoon |
|---|---|---|
| **Cocoon.se (Laurent Bourrelly)** | Pionnier du cocon sémantique FR, méthodologie établie | Outil manuel/Excel, zéro visualisation, pas d'IA |
| **Sitebulb Internal Linking** | Rapport HTML détaillé, PageRank interne | Rapport statique, aucun chat, pas d'auto-link |
| **Ahrefs Internal Links report** | Data à jour + score autorité interne | Vue plate table, pas de graphe interactif, pas d'agent |
| **Neuronwriter Cocoon feature** | Recommandations sémantiques | Pas de visualisation 3D, one-shot, pas de persist session |
| **Thot SEO maillage interne** | Recommandations en français + intégré au workspace | Table/liste seulement, pas de graphe, pas d'IA agent |

**Barre à atteindre / dépasser** : Cocoon est **le seul** avec (1) visualisation 3D + 2D + radial du semantic graph, (2) chat Stratège contextuel sur nodes sélectionnés, (3) Content Architect intégré (draft complet en un clic), (4) auto-linking + bulk + deploy CMS, (5) intent detection Know/Do/Buy/Navigate/Unknown au crawl. **Angle mort** : concurrents en train de rattraper via export MCP → Crawlers doit exposer les données cocoon via MCP (voir #17 Copilot).
