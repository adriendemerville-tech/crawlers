# Audit Brief — Content Architect
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération

## 1. Objectif métier
Produire un plan éditorial + brouillon structuré (H1/H2/H3, meta, images, liens internes, CTA) prêt à publier CMS, mutualisé cross-user via le cache markdown.

## 2. Cas d'usage principaux
- Premium → ouvre `CocoonContentArchitectModal` depuis Cocoon → génère structure + publie
- Premium → clique "Utiliser" sur Recommandation Crawlers (cross-user cache) → parse markdown H1/H2
- Pro Agency → génère batch depuis Workbench → drafts pushés CMS
- Copilot Stratège → skill `generate_content_architecture` → draft dans `cocoon_architect_drafts`

## 3. Logique métier clé
- Pipeline éditoriale 4 étages : briefing → stratège → rédacteur → tonalisateur (routage LLM par domaine × type)
- Backend : `content-architecture-advisor`, `extract-architect-fields`, `_shared/contentArchitectCache.ts`
- Cache : sha256(keyword|page_type|length|lang|h1|h2[]|secondary_keywords[]) — domain EXCLU pour mutualisation
- RPC SECURITY DEFINER : `get_shared_architect_recommendation` (anonymisé, is_shareable=true)
- Publish CMS : `dictadevi-actions` avec garde Markdown→HTML (marked@12)
- Voice DNA injectée dans le prompt tonalisateur
- Contraintes mémoire : `features/content-architect/markdown-cache-fr`, `tech/architect/editorial-pipeline-fr`, `tech/content-architect/voice-dna-fr`, `tech/architect/content-advisor-async-fr`

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |
|---|---|---|
| Cohérence Copilot | skill `generate_content_architecture` déclarable, draft partagé Workbench | `copilot_actions`, `cocoon_architect_drafts` |
| Fonctionnel | draft complet H1+3xH2+images+liens+meta+CTA dans >95% des cas | échantillon 20 runs |
| Performance | p95 < 25s pour un article standard | logs edge |
| Coût LLM | < 8¢ / draft (pipeline 4 étages) | `ai_gateway_usage` |
| Cache hit | > 30% cross-user sur clés populaires | `content_architect_cache.hit_count` |
| Sécurité | RPC ne fuit jamais user_id, RLS sur `cocoon_architect_drafts` | security scan |
| UX | badge cache "Depuis le cache · Nj", bouton Régénérer, palette Crawlers | revue |

## 5. Benchmark concurrentiel
- **SurferSEO / NeuronWriter** — brief éditorial basé SERP, très outillé
- **Frase.io** — brief + rédaction, workflow rapide
- **MarketMuse** — enterprise, scoring topique
- **Clearscope** — audit sémantique premium
- Barre : dépasser Surfer en injectant Voice DNA + cache mutualisé + push CMS 7 plateformes en un clic

## 6. Périmètre d'audit
### In scope
- Backend : `content-architecture-advisor/`, `extract-architect-fields/`, `_shared/contentArchitectCache.ts`, pipeline 4 étages
- Frontend : `CocoonContentArchitectModal`, `ContentArchitectStructurePanel`, `ContentArchitectPromptPanel`, `ContentArchitectPage.tsx`
- Tables : `content_architect_cache`, `cocoon_architect_drafts`
### Hors scope
- Code Architect / Advanced Injection (#4)
- Publish CMS interne (`crawlers-internal-cms-handler` déjà mémoire)

## 7. Signaux à mesurer
- SQL : `select edge_function, avg(total_cost), avg(duration_ms) from ai_gateway_usage where edge_function in ('content-architecture-advisor','extract-architect-fields') group by 1`
- Cache : `select count(*), sum(hit_count), avg(extract(day from now()-created_at)) from content_architect_cache where expires_at > now()`
- Cross-user : `select count(*) from content_architect_cache where is_shareable=true`
- Logs edge : `content-architecture-advisor`, `extract-architect-fields`
