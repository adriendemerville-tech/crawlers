# Audit Brief — cms-push (multi-plateformes)
Date : 2026-07-08 · Auditeur : Senior Crawlers Auditor · Baseline : première itération

## 1. Objectif métier
Garantir la fiabilité, la sécurité et la performance de la chaîne de distribution de contenu (push CMS) vers des plateformes tierces. L'enjeu est de transformer Crawlers en un **hub de diffusion universel** capable d'injecter du contenu optimisé SEO/GEO directement chez le client (WordPress, Dictadevi, Shopify, Odoo, etc.) sans friction technique, tout en assurant l'intégrité des données et la validité des clés API.

## 2. Cas d'usage principaux
- **Connexion CMS** : l'utilisateur enregistre sa clé API via un wizard (Smart CMS Connect) et teste la connexion.
- **Push de contenu** : envoi d'articles, de pages ou de redirections depuis le Content Architect vers le CMS cible.
- **Mise à jour (Patch)** : synchronisation des corrections (HTML, SEO tags) sur des contenus déjà publiés.
- **Publication de brouillons** : passage de l'état 'draft' à 'published' à distance.
- **Scan de contenu** : vérification de l'état du contenu distant (Universal Content Scanner) pour détecter les désynchronisations.

## 3. Logique métier clé
- **Abstraction Multi-plateforme** : gestion via `cms_type` (custom_rest, wordpress, shopify, etc.) et routage vers les edges spécialisées.
- **Gestion des Secrets** : stockage des clés API dans `cms_connections.api_key` avec chiffrement/protection (cf. mem://tech/security/code-and-injection-security-v2-fr).
- **Moteur de Déploiement** : orchestration des flux via Parménion V3 (mem://tech/cms/deployment-engine-v3-fr).
- **Scanner Universel** : logique de comparaison entre le contenu local (`cms_page_content`) et le contenu distant via `universal-content-scanner`.
- **Dictadevi Bridge** : implémentation spécifique pour Dictadevi via `dictadevi-actions` (API REST maison, semantic rings, sitemaps LLM).

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |
|---|---|---|
| Fonctionnel | Taux de succès push > 98% | Logs edge functions (success vs errors) |
| Sécurité | 0 leak de clé API en log | Revue de code `cms-register-api-key` |
| UX | Connexion CMS en < 3 étapes | Test manuel `SmartCmsConnectModal` |
| Performance | Latence push < 5s (hors timeout distant) | Logs edge `cms-patch-content` |
| Intégrité | Pas de contenu tronqué ou mal formatté | Test `universal-content-scanner` |
| Design | Respect palette (violet/or/noir/blanc), pas de bleu IA, boutons sans fond | Revue CSS des wizards CMS |

## 5. Benchmark concurrentiel
- **Surfer SEO** : propose un plugin WP pour synchroniser le contenu. Force : simplicité. Faiblesse : mono-plateforme WP, pas de gestion GEO poussée.
- **RankMath** : excellent pour la gestion des metas et l'indexation instantanée. Force : intégration profonde WP. Faiblesse : limité à WP, pas de "push" depuis un outil externe tiers.
- **TODO** : validation utilisateur requise pour confirmer si Frase ou WriteSonic sont des concurrents plus pertinents sur la partie "push multi-plateformes".
- **Barre à dépasser** : fournir un connecteur universel (Custom REST) avec injection de signaux GEO/LLM (sitemaps-llm, semantic rings) que les plugins classiques ne gèrent pas.

## 7. Signaux à mesurer
- **SQL (Usage & Coût)** :
  ```sql
  select edge_function, count(*), avg(total_tokens), avg(cost_usd)
  from ai_gateway_usage
  where edge_function in ('dictadevi-actions', 'cms-register-api-key', 'cms-patch-content', 'cms-publish-draft', 'universal-content-scanner')
    and created_at >= now() - interval '30 days'
  group by 1;
  ```
- **Logs Edge** : monitoring de `dictadevi-actions` pour vérifier les temps de réponse et les erreurs HTTP renvoyées par le CMS cible.
- **Santé des clés** : count de `cms_connections` avec `is_active = false` ou erreurs de handshake répétées.
