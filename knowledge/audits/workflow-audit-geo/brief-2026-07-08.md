# Audit Brief — Workflow Audit GEO
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération dédiée GEO end-to-end

## 1. Objectif métier
Mesurer et améliorer la visibilité d'un site auprès des moteurs génératifs (ChatGPT, Perplexity, Gemini, Claude) via une chaîne complète : détection bots → vérification → attribution → KPIs → recommandations.

## 2. Cas d'usage principaux
- Premium → active Shield Cloudflare via wizard → voit KPIs 7j après
- Premium → consulte onglet GEO : 7 KPIs bandeau + 4 cartes (Quality, Fan-Out, Drop Detector, Bot Mix)
- Pro Agency → compare fan-out par cluster vs concurrents
- Admin → surveille pipeline cron `cron-geo-pipeline`

## 3. Logique métier clé
- Backend : `geo-kpis-aggregate`, `analyze-geo-*`, `cron-geo-pipeline`, `bot-verification-*`, `fan-out-detection`, `shield-*`
- Attribution : 30j strict + multi-touch pondéré exp(-d/15) + fingerprint SHA-256
- Vérification bots : rDNS + ASN officiels → verification_status/method/confidence_score
- Cron : 04h KPIs, 05h CTR, lundi 06h snapshot (limités Shields actifs)
- Gating : Premium+ (Shield), Free voit KPIs démo
- Contraintes mémoire : `features/geo/shield-and-cron-pipeline`, `bot-human-attribution`, `bot-verification-sprint-a`, `fan-out-detection-strategy-fr`, `geo/geo-kpi-metrics-fr`

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |

## 5. Benchmark concurrentiel
- **Otterly.ai** — leader GEO, tracking mentions LLM
- **Peec.ai / Profound / AthenaHQ** — mesure share of voice LLM
- **HubSpot AI Search Grader** — gratuit, très marketing
- **BrightEdge Generative Parser** — enterprise, cher
- Barre : offrir Shield + attribution bot↔human que personne ne fait, tout en atteignant la qualité de mesure d'Otterly

## 7. Signaux à mesurer
