# Audit Brief — render-page + SSR SEO
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération

## 1. Objectif métier
Servir aux bots (Google, IA, réseaux sociaux) un HTML pré-rendu complet avec meta/JSON-LD/hreflang/breadcrumb pour maximiser indexation SEO et citations GEO, en cachant les rendus 24h dans `prerender_cache`.

## 2. Cas d'usage principaux
- **Googlebot / GPTBot** → GET `?route=/blog/mon-slug` → HTML complet avec BlogPosting + Breadcrumb + FAQ JSON-LD → indexation
- **Utilisateur social share** → `?route=/tarifs` → HTML avec OG tags → Twitter/LinkedIn preview
- **Admin** → publish blog article → SSR régénère au prochain hit (TTL 24h)

## 3. Logique métier clé
- 4 types de routes : static (43 hardcodées), `/blog/<slug>`, `/landing/<slug>`, `/guide/<slug>`
- Cache : `prerender_cache` TTL 24h, clé = `route`, `content_hash` SHA-256 (blog uniquement)
- Détection bot par regex UA (30+ patterns) — actuellement log-only
- JSON-LD : BlogPosting / Article / WebPage + BreadcrumbList + FAQPage (si ≥2 Q/R détectées)
- Dépendances : `blog_articles`, `seo_page_drafts`, `prerender_cache`
- Contraintes : mem://tech/seo/render-page-enrichment-fr, mem://tech/seo/canonical-strategy-fr

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |
|---|---|---|
| Fonctionnel | HTML valide, JSON-LD parsable, canonical présent | rich results test |
| Performance | p95 < 500ms cache hit, < 2s fresh | `X-Prerender` header |
| Coût LLM | 0 (aucun appel LLM) | ai_gateway_usage |
| Sécurité | pas de XSS via markdown | audit escaping |
| SEO | Google Search Console indexation | GSC coverage |

## 5. Benchmark concurrentiel
- **Prerender.io** : SaaS pré-rendu, ~$90/mo, cache CDN, ETag natif → **on est en dessous sur ETag/304**
- **Rendertron** (Google) : headless Chrome, coûteux, pas de JSON-LD auto → **on gagne**
- **Next.js ISR** : revalidation on-demand → **on n'a pas d'invalidation sur update**
- Barre : garder la richesse SEO (JSON-LD complets, FAQ auto, sections neutres) + rattraper ETag/304 + invalidation
