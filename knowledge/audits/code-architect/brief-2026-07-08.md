# Audit Brief — Code Architect
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération

## 1. Objectif métier
Injecter du code (JSON-LD, meta, HTML, scripts) sur un site cible via le bridge IKtracker v4, de façon sécurisée (DOMPurify) et traçable (catalogue d'injections), sans jamais toucher au raw HTML côté user.

## 2. Cas d'usage principaux
- Premium → workbench Code Architect → choisit catégorie injection → prévisualise → déploie via IKtracker
- Pro Agency → batch injections multi-URL depuis Workbench Autopilot
- Admin → audit catalogue d'injections déployées + rollback
- Copilot Félix → skill `inject_code_advanced` en approval mode

## 3. Logique métier clé
- Advanced Injection Workflow : catégories (SEO tags, JSON-LD schema.org, tracking, custom HTML)
- Bridge : `iktracker-bridge-v4` (mémoire dédiée), payload adapté par stratégie
- Sécurité : DOMPurify côté edge + slug memory pour cohérence
- Contrainte : raw HTML/code manual editing disabled côté user (Core memory)
- Stratégie : `script-injection-strategy` module
- Contraintes mémoire : `tech/api/iktracker-bridge-v4-fr`, `tech/architect/script-injection-strategy-fr`, `tech/security/code-and-injection-security-v2-fr`, `tech/architect/advanced-injection-workflow-fr`

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |
|---|---|---|
| Cohérence Copilot | skill `inject_code_advanced` en approval, résultats en vector memory | `copilot_actions.needs_approval` |
| Fonctionnel | injection réussie sur 7 plateformes CMS, rollback fonctionnel | tests manuels |
| Sécurité | DOMPurify obligatoire, aucune injection XSS possible, isolation `auth.uid()` | security scan + tests XSS |
| Performance | p95 injection < 3s | logs edge |
| Coût LLM | < 2¢ / injection assistée | `ai_gateway_usage` |
| UX | preview diff avant push, bouton bordure sans fond, palette Crawlers | revue |
| Traçabilité | 100% injections loggées avec user_id, url, category, checksum | table `code_injections` |

## 5. Benchmark concurrentiel
- **Google Tag Manager** — universel, pas SEO-first, courbe apprentissage forte
- **SEOPress / RankMath / Yoast** — plugins WP, pas multi-CMS
- **Schema App** — spécialiste JSON-LD, cher enterprise
- **Sitebulb / Screaming Frog** — audit uniquement, pas d'injection
- Barre : offrir GTM-like multi-CMS + assistance IA + preview diff, sécurité DOMPurify systématique

## 6. Périmètre d'audit
### In scope
- Backend : edges liées à `iktracker-*`, `script-injection-strategy`, DOMPurify wrapper
- Frontend : Code Architect workbench (composants), Advanced Injection UI
- Tables : `code_injections`, catalogue, slug memory
### Hors scope
- Content Architect (#3)
- CMS custom REST key registration (déjà mémoire)

## 7. Signaux à mesurer
- SQL : `select edge_function, count(*), avg(duration_ms) from ai_gateway_usage where edge_function like 'iktracker%' or edge_function like '%injection%' group by 1`
- Logs : `iktracker-*`, injection edges
- Sécurité : `security--run_security_scan` + revue DOMPurify
- Rollback : rate = `count(rollback)/count(deploy)`
