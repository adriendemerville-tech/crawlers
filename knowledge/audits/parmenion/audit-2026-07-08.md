## #169 · P1 · 2j — Propagation aveugle après skip d'Audit
L'orchestrateur continue les phases Diagnose/Prescribe même si l'Audit est sauté (crawl en cours), travaillant sur des données obsolètes. Fix : Interrompre le cycle dans autopilot-engine si l'orchestrateur retourne un statut skipped sur une phase critique.

## #170 · P0 · 3j — Race condition critique sur la phase Validate
L'enchaînement immédiat de Execute et Validate dans le même thread empêche la détection du déploiement (cache CMS, latence workers). Fix : Découpler Validate pour l'exécuter avec un délai minimum de 30 minutes ou lors du cycle suivant (t+1).

## #171 · P1 · 1j — Évaporation des recommandations stratégiques (Tasks 2-8)
Le stratège génère un plan de 8 tâches mais l'orchestrateur n'extrait et n'exécute que la première, ignorant le reste du plan. Fix : Persister le plan complet dans strategist_recommendations et modifier Execute pour traiter les tâches en attente avant de re-prescrire.

## #172 · P2 · 0.5j — Collision de configuration multi-target sur le domaine
L'utilisation de ilike('domain') sans filtrage strict par tracked_site_id peut retourner une mauvaise config en cas de domaines en doublon (staging/prod). Fix : Harmoniser toutes les requêtes pour utiliser systématiquement le tuple (domain, tracked_site_id) en correspondance exacte.

## #173 · P2 · 2j — Risque de Timeout Lambda sur cycles LLM Sonnet 4.5
L'exécution séquentielle de 5 phases incluant des appels LLM lents peut dépasser les 300s de timeout d'Edge Function. Fix : Refactoriser le loop autopilot-engine pour utiliser une file d'attente (Edge HTTP async) ou augmenter drastiquement le timeout par phase.

## #174 · P1 · 1j — Garde-fou Backlog inopérant sur exécution bloquée
Le guard stoppe la prescription si le backlog est plein (>5) mais ne gère pas les tâches qui échouent en boucle dans la phase Execute. Fix : Implémenter un retry_count sur les décisions et basculer en 'failed' définitif après 3 tentatives infructueuses pour libérer le backlog.

## #175 · P2 · 1j — Instabilité du Persona par rotation multi-phase
La rotation Round-robin risque de changer de persona entre Audit et Execute au sein d'un même cycle si elle est invoquée à chaque phase. Fix : Fixer le persona au niveau du autopilot-engine pour toute la durée du cycle et le transmettre comme constante à l'orchestrateur.

## #176 · P1 · 1j — Faiblesse de l'isolation de sécurité (site_id vs domaine)
L'orchestrateur fait confiance aux paramètres domain et tracked_site_id sans vérifier leur corrélation fonctionnelle en base de données. Fix : Intégrer un guard d'intégrité au démarrage vérifiant l'appartenance du domaine au tracked_site_id et à l'utilisateur appelant.
