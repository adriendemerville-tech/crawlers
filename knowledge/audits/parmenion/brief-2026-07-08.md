# Audit Brief — Parménion (autopilot orchestrator)
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération

## 1. Objectif métier
Orchestrateur stratégique autonome qui pilote les sites en Autopilot : audit → diagnose → prescribe → execute (CMS) → validate. Publie du contenu SEO/GEO original sans intervention humaine, respecte les gardes (backlog, cannibalisation, saturation, diversité personas).

## 2. Cas d'usage principaux
- **Cron 6h** → `autopilot-engine` réveille chaque site actif → invoque `parmenion-orchestrator` → phase suivante
- **Prescribe** → génère 8 tâches déterministes via `cocoon-strategist` (mem `parmenion-v3-fr`)
- **Execute** → `iktracker-actions` / `dictadevi-actions` / `wpsync` selon domaine
- **Validate** → `autopilot-validate-deployed` re-scanne HTML pour vérifier déploiement
- **Feedback** → `parmenion-feedback` remonte les résultats au workbench

## 3. Logique métier clé
- **5 phases** (audit → diagnose → prescribe → execute → validate) — no rollback
- **Backlog guard** : pause auto si >5 décisions CMS planned non exécutées (mem `backlog-guard-fr`)
- **Cluster diversity** : SQL cap 2 items/cluster + topic exploration map (mem `cluster-diversity-engine-fr`)
- **Persona rotation** : round-robin round-robin entre personas décomposés (mem `persona-decomposition-engine-fr`)
- **Dedup layer E** : Jaccard synonym-aware + saturation guard (mem `dedup-synonym-layer-fr`)
- **Anti-hallucination gates** (mem `anti-hallucination-gates-fr`)
- **Auto-config on target** : trigger DB crée `autopilot_configs` dry_run à chaque insert `parmenion_targets` (mem `auto-config-on-target-fr`)
- **Crawl guard** : skip cycle si crawl en cours (planif dans `cron-crawl-scheduler`, mem `crawl/scheduler-fr`)
- **Modèles** : Stratège Gemini 3.1 Pro → Writer Sonnet 4.5 → Tonalizer Gemini 3.5 Flash (mem `combo-abc-allocation-fr`)
- **Tables** : `parmenion_targets`, `parmenion_decision_log`, `autopilot_configs`, `architect_workbench`, `persona_rotation_log`, `saturation_snapshots`, `strategist_recommendations`

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |
|---|---|---|
| Fonctionnel | 5/5 phases exécutées sans erreur sur 90% des cycles | `parmenion_decision_log.status` |
| Performance | Cycle < 5 min p95 | `execution_time` dans decision_log |
| Coût LLM | < 5¢ / cycle prescribe (Sonnet 4.5 + caching) | `ai_gateway_usage` filtré `edge_function` |
| Sécurité | 100% appels LLM via `aiGatewayFetch`, admin-only route | Grep + test 403 |
| Qualité contenu | Jaccard < 0.5 vs existant, diversité cluster respectée | `saturation_snapshots` |

## 5. Benchmark concurrentiel
| Concurrent | Force | Faiblesse vs Parménion |
|---|---|---|
| **SEO.ai** | Auto-génération + publish WordPress, UI simple | Pas de garde saturation ni cluster diversity, pas de persona rotation, pas de validate post-deploy |
| **Frase autopilot** | Brief + writer intégré | One-shot par article, aucune vision pluri-site multi-cycle |
| **Byword.ai** | Publication massive automatisée WordPress | Modèle single-shot, pas d'anti-cannibalisation, pas de re-crawl validate |
| **AutoblogAI** | Pilote automatique blog complet | Zéro garde sémantique, saturation garantie à moyen terme |
| **Stratège humain freelance** | Jugement contextuel fin | Non scalable, cher, pas de cycle continu 6h |

**Barre à atteindre / dépasser** : Parménion est **le seul autopilot SEO/GEO** avec (1) validation post-déploiement via re-crawl, (2) garde saturation + cluster diversity + persona rotation, (3) pipeline 5 phases déterministe, (4) backlog guard qui pause automatiquement, (5) mémoire pluri-cycle via `parmenion_decision_log`.

## 7. Signaux à mesurer
```sql
-- Coût LLM Parménion 30j
select edge_function, model, count(*), sum(total_credits)
from ai_gateway_usage
where edge_function like 'parmenion%' or edge_function = 'autopilot-engine'
  and created_at > now() - interval '30 days'
group by 1,2 order by 4 desc;

-- Taux réussite cycles
select pipeline_phase, status, count(*)
from parmenion_decision_log
where created_at > now() - interval '30 days'
group by 1,2;

-- Backlog par site
select domain, count(*) as pending_actions
from parmenion_decision_log
where status = 'planned' group by 1 having count(*) > 5;
```
