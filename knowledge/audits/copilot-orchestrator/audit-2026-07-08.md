# Audit — copilot-orchestrator
Date : 2026-07-08 · Auditeur : Lovable (subagent)
Baseline brief : `brief-2026-07-08.md`

## Synthèse
Architecture solide (séparation Félix/Stratège, boucle agent SSE streaming parallèle). Points critiques identifiés sur le rappel mémoire vectoriel (RPC en fallback JS) et la charte graphique (bleu IA sur Félix).

## Findings

## #159 · P0 · 1j — Recall mémoire vectorielle hybride : dégradation en fallback JS
`recallMemoryContext` tombe sur le fallback JS (cosine 50 lignes) car la RPC hybride échoue avec le service role (`auth.uid()` null) et `p_user_id` n'est pas passé. Recall cross-session inopérant ou dangereux.
Fix : Passer explicitement `p_user_id` à `search_copilot_turns_hybrid` (SECURITY DEFINER) avec filtrage interne.

## #160 · P1 · 1j — Violation charte "Pas de bleu IA" sur Félix
`AgentChatShell.tsx` L188 utilise `border-primary bg-primary` pour les bulles Félix. `primary` = bleu IA générique, contraire au branding Crawlers.
Fix : Bascule Félix en gris anthracite (`bg-neutral-800`) ou noir Crawlers, Stratège garde l'accent safran.

## #161 · P1 · 1j — Anthropic cache breakpoints manquants sur l'historique
`callLLM` active le cache Anthropic uniquement sur le system prompt. Pas de `cache_control` sur les messages historique → re-lecture payante à chaque itération de la boucle agent.
Fix : Injecter `cache_control: { type: 'ephemeral' }` sur le dernier message user du tour précédent dans `loadHistory`.

## #162 · P2 · 0.5j — Skill `trigger_audit` autorisée pour Félix (incohérence rôle)
Félix (SAV gratuit) peut proposer `trigger_audit` en approval → friction UX + risque conseil payant.
Fix : Passer `trigger_audit` en `forbidden` pour Félix dans `personas.ts`, réservée au Stratège.

## #163 · P2 · 1j — Séquençage inefficace des skills navigation
`runAgentLoopStreaming` L1258 traite `navigate_to` / `open_audit_panel` séquentiellement alors qu'elles sont sans effet de bord serveur.
Fix : Déplacer les skills `navigate` dans le batch `parallelizable` pour réduire TTFB.

## #164 · P1 · 1.5j — safeServiceCall manquant sur `cms_publish_draft` (cas blog)
`registry.ts` L404 écrit sur `blog_articles` via service role en vérifiant uniquement `admin`, sans validation périmètre.
Fix : Wrapper CMS avec vérification granulaire de propriété/permission même pour admins.
