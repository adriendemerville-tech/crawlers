# Audit Brief — copilot-orchestrator
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération

## 1. Objectif métier
Fournir un agent conversationnel (Félix SAV + Stratège) capable d'exécuter des skills métier (audit, live search, market diagnosis, escalation, mémoire vectorielle) au nom de l'utilisateur Crawlers, avec approbation humaine sur les actions sensibles et scoring qualité automatique.

## 2. Cas d'usage principaux
- **Free** → pose une question SAV → Félix répond, escalade si besoin (skill `escalate_to_phone` avec approval)
- **Premium** → demande un diagnostic marché → Stratège exécute `market_diagnosis` (4 appels DataForSEO parallèles, quota 3/j)
- **Pro Agency** → audit maillage interne → skill `audit_internal_mesh` sur cocoon_auto_links
- **Admin (mode créateur)** → skills admin (lookup, creator-skills) via `personas.ts`
- **Tout plan** → recall mémoire vectorielle hybride (pgvector + BM25) sur turns précédents

## 3. Logique métier clé
- **Agent loop** : `copilot-orchestrator/index.ts` — boucle streaming SSE, `stepCountIs(50)`, tool calls parallèles
- **Personas** : `_shared/copilot/personas.ts` (Félix Haiku 4.5, Stratège Sonnet 4.5) via `combo-abc-allocation-fr`
- **Skills registry** : `_shared/copilot/skills/registry.ts` — policies auto / approval / forbidden par persona
- **Sécurité prompt** : wrappers `<user_input>` / `<tool_result>` + neutralisation balises (zero-width space) — mem `prompt-safety-layer`
- **Anti-bypass RLS** : `safeServiceCall` — mem `p0-hardening-safeservicecall-session-status`
- **Session status** : flag `processing` avec reconciliation 90s
- **Mémoire vectorielle** : embed `_user_message` uniquement, worker cron 5min, RPC `search_copilot_turns_hybrid` (0.6 cosine + 0.4 BM25)
- **Live search** : quotas Free 3/conv, Premium 5/conv, Pro 20/jour
- **Scoring** : upsert auto dans `sav_quality_scores` par tour
- **Tables** : `copilot_sessions`, `copilot_actions`, `sav_quality_scores`, `ai_gateway_usage`
- **Contraintes mémoires** : `combo-abc-allocation-fr`, `prompt-safety-layer`, `vector-memory-hybrid-recall`, `p0-hardening-safeservicecall-session-status`, `p0-loadhistory-batch-stop-draft-coherence`, `live-search-bloc2`, `escalation-scoring-bloc3`, `market-diagnosis-page-targeting-v2`

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |
|---|---|---|
| Fonctionnel | Toutes skills accessibles selon policy persona, approval respecté, memory recall injecté | Test manuel + logs |
| Performance | Premier token < 1.5s (SSE), tool call < 3s p95 | `edge_function_logs copilot-orchestrator` |
| Coût LLM | Félix < 0.5¢/tour · Stratège < 3¢/tour (avec cache Anthropic) | `ai_gateway_usage` filtré `run_id` |
| Sécurité | 100% appels via `safeServiceCall`, 100% prompts wrappés, 0 skill accessible hors policy | Grep + tests |
| UX | Streaming SSE fluide, boutons approval sans fond (bordure+texte), pas d'emoji, pas de bleu IA | Revue AgentChatShell |

## 5. Benchmark concurrentiel

| Concurrent | Force | Faiblesse vs Crawlers copilot |
|---|---|---|
| **Ahrefs AI (Content Helper + brand mentions)** | Data backlinks/keywords massive, brand mentions AI récentes | Chat one-shot par outil, pas d'agent multi-skills, pas de mémoire conversationnelle cross-session, pas d'approval humain sur actions |
| **Screaming Frog MCP** (v24, mars 2026) | Expose le crawler SF à Claude/LM Studio via MCP — vrai agent qui exécute crawls et exports | Local desktop only, techno crawl pur (pas de GEO/LLM visibility/SAV), pas de personas, pas de scoring qualité, pas de mémoire vectorielle |
| **Thot SEO (FR)** | Optimisation sémantique française, workspace éditeur, prompts IA intégrés | Assistant statique (prompts pré-remplis), pas d'agent exécutant, pas d'orchestration transverse, pas de SAV |
| **Mentionable (FR)** | Tracking GEO 7 LLMs + recommandations IA + MCP plug into ton IA | Focus GEO tracking uniquement, pas d'exécution (audit/publish/escalation), pas de persona SAV, pas de skills métier composables |
| **Rank.ai (+ MCP)** | Plateforme SEO all-in-one + MCP conversationnel pour audits via Claude/Cursor | MCP = read-only (query reports), pas d'agent actionneur, pas de gating par plan, pas de mémoire vectorielle hybride |

**Barre à atteindre / dépasser** : Crawlers est aujourd'hui le seul à combiner **(1)** agent multi-skills exécutant (audit + publish + escalation + market diagnosis), **(2)** deux personas différenciés (Félix SAV Haiku 4.5 / Stratège Sonnet 4.5), **(3)** mémoire vectorielle hybride pgvector+BM25 cross-session, **(4)** approval humain policy-based par persona, **(5)** scoring qualité auto par tour. **Angle mort à surveiller** : Screaming Frog MCP et Rank.ai MCP démocratisent l'accès agent à leurs outils depuis Claude — Crawlers doit exposer son copilote via **MCP externe** pour rester compétitif (le manifest `mcp-manifest.json` existe, à auditer).

## 7. Signaux à mesurer
- SQL coût :
  ```sql
  select model, count(*), sum(total_credits) 
  from ai_gateway_usage 
  where edge_function = 'copilot-orchestrator' 
    and created_at > now() - interval '30 days'
  group by model order by 3 desc;
  ```
- SQL volume : `select count(*) from copilot_sessions where created_at > now() - interval '30 days'`
- SQL scoring : `select avg(score), count(*) from sav_quality_scores where created_at > now() - interval '30 days'`
- Logs edge : `edge_function_logs copilot-orchestrator` (chercher erreurs, timeouts, `safeServiceCall` violations)
- Sécu : `security--run_security_scan` filtré sur les 3 tables du périmètre
