# Audit Brief — audit-matrice (+ parse-matrix-* + MatricePrompt)
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération

## 1. Objectif métier
Permettre à l'utilisateur de **comparer N URLs sur M critères SEO/GEO** dans une matrice pivotable (heatmap, cube 3D, delta) — soit à partir d'un prompt (`MatricePrompt`), soit à partir d'un import DOCX/CSV (`ImportStepper`). Cœur du storytelling "benchmark concurrentiel" Crawlers.

## 2. Cas d'usage principaux
- **Free** → 1 matrice / mois, max 3 URLs × 5 critères.
- **Premium** → matrices illimitées, jusqu'à 10 URLs × 20 critères, export CSV, cube 3D.
- **Pro Agency** → matrices multi-clients, comparaison delta entre deux dates (`MatriceCompare`), snapshot historique.
- **Admin** → voit toutes les matrices, débogue les parsers, corrige les schémas d'affichage (`matrix_display_schemas`).

## 3. Logique métier clé
- **Deux voies d'entrée** :
  - **Prompt** (`MatricePrompt.tsx` 1 639 l. — le plus gros fichier front du projet) → génère matrice via LLM
  - **Import** (`ImportStepper.tsx` 720 l.) → upload DOCX/CSV → parse via 3 parsers spécialisés (`parse-doc-matrix` 368 l., `parse-matrix-hybrid` 384 l., `parse-matrix-geo` 734 l.)
- **Backend** :
  - `audit-matrice/index.ts` (499 l.) — orchestrateur
  - 3 parsers → duplication suspectée (fusion en un parser paramétrable ?)
- **Dépendances** :
  - LLM : Lovable AI via `aiGatewayCall` — **à vérifier**
  - Tables : `audit_matrix_sessions`, `audit_matrix_results`, `matrix_audit_sessions`, `matrix_audit_results`, `matrix_audits`, `matrix_display_schemas`, `matrix_errors`, `prompt_matrix_imports`, `prompt_matrix_items` → **7 tables très proches, redondance schéma suspectée**
  - Mémoires applicables : `matrix-system-v2`, `unified-requirements-matrix`, `combo-abc-allocation`
- **Gating** : plan-based quotas via `useAdmin`/`profiles.plan`.
- **Visualisations** : Cube 3D (`MatriceCube3D` 358 l., `BenchmarkCube3D` 387 l.), heatmap (`BenchmarkHeatmap` 362 l.), pivot (`MatricePivotView` 304 l.), delta (`MatriceDeltaView` 160 l.).

## 5. Benchmark concurrentiel
- **Ahrefs Content Gap** — compare 3 concurrents sur mots-clés partagés. Force : data. Faiblesse : 1 seul axe (keywords), pas de matrice libre.
- **Semrush Keyword Gap** + **Backlink Gap** — même logique, 2 axes. Faiblesse : idem.
- **Screaming Frog Custom Extraction** — matrice URL × attribut extrait, expert only, aucun scoring auto. Faiblesse : brut, no viz.
- **Thot SEO / SEO Rank** — pas d'équivalent matriciel.
- **Mentionnable** — mentions LLM par requête × plateforme, matrice implicite mais 2D fixe. Faiblesse : pas de custom.
- **Barre à dépasser** : **seule matrice N×M totalement libre** (URLs × critères définissables par prompt OU import), avec **cube 3D pivotable** unique sur le marché, delta temporel (`MatriceCompare`) et scoring auto. Différenciateur radical vs Content Gap / Keyword Gap qui restent 2D et fermés.

## 7. Signaux à mesurer

  ```sql
  select edge_function, count(*), avg(total_tokens), avg(cost_usd)
  from ai_gateway_usage
  where edge_function in ('audit-matrice','parse-doc-matrix','parse-matrix-hybrid','parse-matrix-geo')
    and created_at >= now() - interval '30 days'
  group by 1;
  ```
