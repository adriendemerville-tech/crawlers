# Audit — `conversion-optimizer` · 2026-07-08-bis
**Auditeur** : Lovable · **Périmètre** : `analyze-ux-context/index.ts` (1 334 l.), `agent-ux/index.ts`, `src/pages/ConversionOptimizer.tsx` (848 l.), `AnnotatedPageView.tsx` (478 l.), `CROReportPreviewModal.tsx` (370 l.), `ManualAnnotationOverlay.tsx` (264 l.), table `ux_context_analyses`, bucket `ux-screenshots`.

## Résumé exécutif

Le combo screenshot annoté + GA4 behavioral + voice_dna reste différenciant. La logique métier est solide et bien découpée (3 modes séparés, tracking coût présent, RLS de base en place). **Deux bugs critiques bloquent la fiabilité en production** : le sémaphore Browserless existe mais n'est jamais importé, et le modèle LLM référencé (`gemini-3-flash-preview`) n'existe pas sur l'AI Gateway. Le bucket de screenshots est public sans expiration — fuite de données cross-user par URL guessing. Le monolithe 1 334 l. rend la maintenance difficile.

## Findings P0 — Bloquant production

### P0-1 · Sémaphore Browserless non branché
**Fichier** : `supabase/functions/analyze-ux-context/index.ts` — aucun import de `browserlessSemaphore.ts`
**Preuve** : `grep -n "withBrowserlessSlot\|browserlessSemaphore" analyze-ux-context/index.ts` → zéro résultat. Pourtant `_shared/browserlessSemaphore.ts` est présent et fonctionnel (MAX_CONCURRENT=1, WAIT_TIMEOUT=30 s).
**Impact** : En charge simultanée, N isolates ouvrent chacun 2 sessions Browserless (screenshot + findTextPositions) → risque de 429 Browserless immédiat, coût non maîtrisé.
**Fix** :
```ts
// index.ts — remplacer appels directs fetch Browserless par :
import { withBrowserlessSlot } from '../_shared/browserlessSemaphore.ts';
const screenshotResult = await withBrowserlessSlot(
  () => captureScreenshotWithAnnotations(page_url, tracked_site_id, serviceClient),
  'ux-screenshot',
);
// idem pour findTextPositions
```
**Effort** : 30 min · **Score** : 2/10

---

### P0-2 · Modèle LLM inexistant — silences AI garantis
**Fichier** : `supabase/functions/analyze-ux-context/index.ts:124,372,473,512`
**Preuve** : `model: 'google/gemini-3-flash-preview'` — ce slug n'existe pas sur OpenRouter/AI Gateway. Le modèle courant est `google/gemini-2.5-flash-preview-05-20` ou `google/gemini-flash-1.5`. Si l'AI Gateway renvoie 404 le handler retourne `jsonError('AI analysis failed', 500)` sans retry.
**Impact** : Toute analyse default ou manual-suggestions échoue silencieusement si l'AI Gateway valide strictement le nom de modèle.
**Fix** : Remplacer les 3 occurrences par le slug validé ; centraliser dans une constante :
```ts
const LLM_MODEL = 'google/gemini-2.5-flash-preview-05-20'; // à confirmer via AI Gateway catalog
```
**Effort** : 15 min · **Score** : 1/10

---

### P0-3 · Timeout Browserless 45 s > timeout page.goto recommandé
**Fichier** : `supabase/functions/analyze-ux-context/index.ts:674,874`
**Preuve** : `timeout: 45000` dans `page.goto` dans les deux scripts Browserless. La mémoire `mem://tech/browserless/timeout-and-semaphore-fix-fr` préconise 30 s. Avec scroll progressif + lazy-load (≈ 10 s) la fenêtre utile est 30–35 s max.
**Impact** : Une page lente provoque un timeout Browserless côté cloud avant que le script ne rende la main → pas de screenshot, mais on a déjà payé le slot. Le mode `annotations-only` rechante une 2ᵉ session (positions) sans screenshot — doublement du risque.
**Fix** : Réduire `timeout: 30000` dans les deux scripts ; wrapper dans sémaphore (P0-1) pour éviter le cumul de sessions.
**Effort** : 15 min · **Score** : 2/10

## Findings P1 — Sécurité / correctness

### P1-1 · Bucket `ux-screenshots` public sans TTL — fuite cross-user par URL guessing
**Fichier** : `supabase/migrations/20260409135457_*.sql:3` — `public: true`
**Preuve** : Toute URL `https://<project>.supabase.co/storage/v1/object/public/ux-screenshots/<tracked_site_id>/<timestamp>.jpg` est accessible sans authentification. Le path contient `tracked_site_id` (UUID) mais pas `user_id` — un concurrent connaissant le site_id peut énumérer les screenshots par timestamp.
**Impact** : Fuite de captures d'écran pleine page potentiellement sensibles (données tarifaires, contenu privé stagingé).
**Fix** : Passer le bucket en `public: false` + signer les URLs avec `createSignedUrl` (TTL 1 h) ; stocker le chemin `fileName` dans `ux_context_analyses` plutôt que l'URL publique :
```ts
const { data: signedData } = await serviceClient.storage
  .from('ux-screenshots')
  .createSignedUrl(fileName, 3600);
```
**Effort** : 2 h · **Score** : 3/10

---

### P1-2 · Pas de politique RLS UPDATE sur `ux_context_analyses`
**Fichier** : `supabase/migrations/20260409133132_*.sql:16-28`
**Preuve** : 3 policies (SELECT / INSERT / DELETE) sur `auth.uid() = user_id` — aucune UPDATE. Le code utilise `serviceClient.update()` (bypass RLS) ce qui fonctionne mais supprime toute protection si un futur appel utilise `userClient`.
**Impact** : Potentielle mise à jour cross-user des annotations si le refactoring change le client.
**Fix** :
```sql
CREATE POLICY "Users can update own ux analyses"
ON public.ux_context_analyses FOR UPDATE
USING (auth.uid() = user_id)
WITH CHECK (auth.uid() = user_id);
```
**Effort** : 10 min · **Score** : 8/10

---

### P1-3 · Absence de cache — re-payer LLM + Browserless sur analyse identique
**Fichier** : `supabase/functions/analyze-ux-context/index.ts` — aucun `SELECT` d'une analyse récente avant de lancer
**Preuve** : Aucun lookup `ux_context_analyses WHERE page_url = $1 AND created_at > now() - interval '1 day'` avant l'appel LLM.
**Impact** : Chaque clic "Analyser" re-paye Browserless (~0.3–0.5 ¢) + LLM (~1–2 ¢) même si le contenu n'a pas changé.
**Fix** : Ajouter un TTL configurable (défaut 24 h) ; invalider si `crawl_pages.updated_at` est postérieur à `ux_context_analyses.created_at` :
```ts
const { data: cached } = await serviceClient
  .from('ux_context_analyses')
  .select('*')
  .eq('tracked_site_id', tracked_site_id)
  .eq('page_url', page_url)
  .gt('created_at', new Date(Date.now() - 86_400_000).toISOString())
  .order('created_at', { ascending: false })
  .limit(1)
  .maybeSingle();
if (cached) return jsonOk({ ...cached, cached: true });
```
**Effort** : 1 h · **Score** : 5/10

### P1-4 · Double tracking LLM en mode `default`, aucun en `manual-suggestions`
**Fichier** : `supabase/functions/analyze-ux-context/index.ts:473-474` vs `168`
**Preuve** : Mode `default` → `trackTokenUsage` + `logAIUsageFromResponse` (doublon). Mode `manual-suggestions` → seulement `logAIUsageFromResponse` (`trackTokenUsage` absent). Asymétrie non intentionnelle.
**Fix** : Extraire un helper `trackLLMUsage(serviceClient, model, fn, usage)` appelé uniformément dans les 3 modes.
**Effort** : 30 min · **Score** : 6/10

---

### P1-5 · Monolithe 1 334 lignes — 3 modes dans un handler unique
**Fichier** : `supabase/functions/analyze-ux-context/index.ts:1-1334`
**Preuve** : Les modes `default` / `annotations-only` / `manual-suggestions` partagent le même handler + 3 helpers DOM inline (800 l. de scripts Browserless en template string).
**Impact** : Tests unitaires impossibles, temps de lecture élevé, risque d'effets de bord entre modes.
**Fix** : Découpe naturelle → `handleDefaultAnalysis()`, `handleAnnotationsOnly()`, `handleManualSuggestions()` dans des fichiers séparés `_modes/`; extraire les scripts Browserless dans `_scripts/screenshot.ts` et `_scripts/findPositions.ts`.
**Effort** : 4 h · **Score** : 4/10
