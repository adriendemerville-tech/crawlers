# Audit Brief — Crawl multi-pages
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération (audit render-page ciblé fait le 2026-07-08 en parallèle, à ré-utiliser)

## 1. Objectif métier
Découvrir toutes les URLs d'un site (mode `detect` gratuit) puis scraper N pages (mode `analyze`, consomme les crédits) pour extraire titre/meta/H1-H6/schema.org/liens/images/anchors, calculer un score SEO par page, détecter les duplicates, et alimenter en aval le keyword_universe, la voice tone et les recommandations Cocoon/Parménion. C'est le **cœur produit** : sans crawl fiable, aucune feature aval ne fonctionne.

## 2. Cas d'usage principaux
- **Free** → détecte ses pages, lance un crawl 10 pages, voit le score global et 3 recos IA
- **Premium** → crawl 50-150 pages, comparaison avec crawl précédent, export sitemap
- **Pro Agency** → crawl 200-500 pages sur plusieurs sites clients, planification cron (`site_crawl_schedule`)
- **Admin** → monitore les jobs (`CrawlManagement.tsx`, `ScannedUrlsRegistry.tsx`), relance les échecs
- **Système** → Parménion / cron-crawl-scheduler déclenchent des re-crawls incrémentaux (full 15j + targeted 5j)

## 3. Logique métier clé
- **Pricing crédits** (`SiteCrawl/types.ts::getCreditCost`) : 5 (≤50p) / 10 (≤100p) / 15 (≤200p) / 25 (≤350p) / 40 (>350p)
- **2 phases** : `detect` (sitemap + Spider map + CMS scan, gratuit) → `analyze` (scraping payant)
- **Cascade scraping** (`_shared/crawlQueue/scraperStrategy.ts`) : renderPage (interne gratuit) → Spider.cloud (payant primaire) → Firecrawl (fallback payant) → Browserless (SPA only, sémaphore 7 slots)
- **SPA probe** : thin content <50 mots OU shell-like title → force Browserless
- **Passe 2 link discovery** : +200 URLs max au-delà du plan initial
- **Crawl incrémental** : réutilisation pages <12h (mémoire `comprehensive-engine-v5-fr`)
- **Finalisation** : BFS depths, duplicates (title/meta/hash), AI summary Gemini Flash, feed keyword_universe, trigger voice tone
- **Score SEO** : `computePageScore` dans `htmlAnalyzer.ts` (seuils UI 160/120 dans types.ts)
- **Tables** : `site_crawls` (18 col, 5 pol), `crawl_pages` (43 col, 3 pol, **32M updates identifiés Vague 1**), `crawl_jobs` (18 col, 3 pol), `site_crawl_schedule` (13 col, 2 pol), `crawl_page_backlinks`, `crawl_index_history`
- **Contraintes projet** :
  - mem://tech/crawl/comprehensive-engine-v5-fr (2 phases)
  - mem://tech/crawl/process-crawl-queue-architecture (modularisation 5 modules)
  - mem://tech/crawl/scheduler-fr (cron quotidien full 15j + targeted 5j)
  - mem://tech/browserless/timeout-and-semaphore-fix-fr (timeouts 30s, sémaphore 7)

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |
|---|---|---|
| Fonctionnel | detect → analyze → finalize sans job bloqué en `processing` >5min | requête `crawl_jobs where status='processing' and updated_at < now()-interval '5min'` |
| Performance | 50 pages crawlées en < 90s (p95) | `crawl_jobs.completed_at - started_at` |
| Coût scraping | < 0.5¢ / page en moyenne (renderPage-first) | ratio `renderPage` vs `spider` vs `firecrawl` vs `browserless` dans logs |
| Coût LLM | AI summary Gemini Flash < 0.2¢ / crawl | `ai_gateway_usage where edge_function='process-crawl-queue'` |
| DB write pressure | pas de retour sur les 32M updates crawl_pages | `pg_stat_user_tables` |
| Sécurité | RLS `auth.uid()` sur toutes tables `crawl_*` | `security--run_security_scan` |
| Modularité | pas de fichier >500 lignes hors `crawl-site/index.ts` (déjà 690) | `wc -l` |
| Robustesse | reprise après crash via checkpoint `crawl_pages` (déjà en place) | test manuel : kill worker mi-parcours |
| UX | boutons bordure+texte (design system Crawlers), pas d'emoji, pas de bleu IA | revue `src/pages/SiteCrawl.tsx` |

## 5. Benchmark concurrentiel
- **Screaming Frog** (desktop, £199/an) : référence absolue, 500 URLs gratuit, filtre par regex, custom extraction XPath/CSS, mais **desktop-only, pas de cron, pas d'IA, pas de comparaison temporelle native**
- **Sitebulb** (£13.5/mois) : concurrent direct de Screaming Frog + rapports auto, **pas d'IA, pas de scoring GEO**
- **Ahrefs Site Audit** : crawl cloud + scheduling + comparaison snapshots, **~99$/mois min, pas de custom LLM summary, pas de détection SPA fine**
- **Semrush Site Audit** : idem Ahrefs, 130 checks, **coût crédits opaque, pas de détection SPA avec Browserless**
- **Oncrawl** (entreprise, 249€/mois+) : le plus proche techniquement (log analyzer + BFS depth + segments), **overkill et cher pour PME**

**Barre à atteindre / dépasser** : seul crawler qui combine **détection gratuite + scraping cascade 4 sources (avec SPA auto-probe) + AI summary + comparaison snapshots incrémentale + scoring SEO/GEO unifié + planification cron** — le tout à un pricing crédits transparent (5-40 crédits par crawl). Aucun concurrent grand public ne fait la cascade renderPage→Spider→Firecrawl→Browserless.

## 7. Signaux à mesurer
- SQL :
  - `select edge_function, count(*), sum(cost_credits) from ai_gateway_usage where edge_function in ('crawl-site','process-crawl-queue') group by 1`
  - `select status, count(*), avg(extract(epoch from (completed_at - started_at))) as avg_sec from crawl_jobs where created_at > now() - interval '7 days' group by 1`
  - `select provider, count(*) from paid_api_calls where edge_function='process-crawl-queue' and created_at > now() - interval '7 days' group by 1` → ratio renderPage/Spider/Firecrawl/Browserless
  - `select count(*), sum(pg_column_size(*)) from crawl_pages` → confirmer volumétrie 32M
- Logs edge : `supabase--edge_function_logs crawl-site`, `process-crawl-queue`, `cron-crawl-scheduler`
- Slow queries : `supabase--slow_queries` filtre `crawl_pages`
- Front : bundle size `SiteCrawl.tsx` + hooks, re-renders (le fichier probablement gros)
- Signaux qualité code : `console.log` count, `: any` count, doublons entre `crawl-site` et `strategic-crawl`
