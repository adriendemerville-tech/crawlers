# Audit Brief — Conversion Optimizer (Conversion Audit)
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération

## 1. Objectif métier
Fournir à l'utilisateur une analyse UX/CRO contextuelle d'une page crawlée (score 8 axes + suggestions actionnables + screenshot annoté) pour identifier les frictions de conversion — sans re-crawl (réutilise `crawl_pages`).

## 2. Cas d'usage principaux
- **Free / Premium** → sélectionne un site + une page → clic "Analyser" → obtient 8 scores + N suggestions ancrées visuellement sur screenshot pleine page
- **Premium+** → dessine des annotations manuelles sur le screenshot → recalcule suggestions ciblées (mode `manual-suggestions`) → push vers `architect_workbench`
- **Historique** → réhydrate une analyse ancienne (mode `annotations-only`) pour re-tracer les bulles sur screenshot existant
- **Landing** (`/conversion-optimizer` public 313 l.) → lead magnet

## 3. Logique métier clé
- **Axes** : `tone`, `cta_pressure`, `alignment`, `readability`, `conversion`, `mobile_ux`, `keyword_usage`, `chunkability` (8 axes)
- **Sources** : `crawl_pages` (contenu), `tracked_sites` (business_type, voice_dna, target_audience…), `keyword_universe` (top keywords), `ga4_behavioral_metrics` (comportement), `content_requirements_matrix` (croMatrix par page_type)
- **Screenshot** : Browserless full-page → extraction bounding boxes via `findTextPositions` (matching texte suggestion → coordonnées DOM)
- **Modes** : `default` (analyse complète LLM + screenshot), `annotations-only` (re-trace bulles), `manual-suggestions` (LLM ciblé sur zones dessinées + push workbench)
- **Persistence** : `ux_context_analyses` (14 col, 3 pol) — screenshot_url, annotations, suggestions, scores, global_score
- **Tracking coût** : `trackPaidApiCall` + `trackTokenUsage` + `logAIUsageFromResponse` (imports présents — appels à vérifier)
- **Contraintes projet** : mem://tech/conversion-optimizer/comprehensive-v3-fr (référencée dans l'index mais **fichier manquant** — TODO), mem://tech/architecture/data-integration-v5-fr (GA4 behavioral), mem://tech/browserless/timeout-and-semaphore-fix-fr (sémaphore 7 slots, timeout 30s)

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |
|---|---|---|
| Fonctionnel | 8 axes retournés + suggestions ancrées | test manuel sur 1 URL |
| Performance | latence p95 < 45s (Browserless 30s + LLM ~10s) | logs edge |
| Coût LLM | < 2¢ / analyse (Gemini 2.5 flash) | `ai_gateway_usage` |
| Sécurité | RLS `ux_context_analyses` sur `auth.uid()`, pas de fuite screenshot cross-user | `security--run_security_scan` |
| UX | Annotations alignées au pixel près sur screenshot, mode dessin fluide | revue visuelle |
| Modularité | Aucun fichier > 500 lignes | wc -l |

## 5. Benchmark concurrentiel
- **Hotjar / Clarity** : heatmaps + session replay — force : data réelle utilisateurs, faiblesse : aucune reco IA contextuelle, aucun scoring axes, requiert tag JS
- **VWO / Optimizely** : A/B testing + suggestions — force : test scientifique, faiblesse : coût élevé (500€+/mois), requiert intégration lourde
- **Landbot / Unbounce Smart Traffic** : optimisation landing via ML — force : automatisation, faiblesse : locked-in plateforme
- **Fibr.ai / Convert Experiences** : audit CRO IA — force : reco contextuelles, faiblesse : anglophone, pas de screenshot annoté visuel
- **Barre à dépasser** : le **screenshot pleine page + bulles ancrées sur suggestions + intégration GA4 behavioral + voice_dna** est un **combo unique** — Hotjar montre le comportement, Fibr donne des reco génériques ; Crawlers fait les deux en un seul écran, sans tag JS, en réutilisant le crawl existant.
