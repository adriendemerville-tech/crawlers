# Brief d'audit : Skills Copilot Post-Orchestrator (Juillet 2026)

## 1. Objectif & Périmètre
Audit de sécurité et de performance des nouveaux skills intégrés au Copilot suite au déploiement de l'orchestrateur central. Le périmètre inclut l'analyse des capacités de recherche temps réel, de diagnostic de marché et de gestion de l'escalade humaine.

- **Skills ciblés** :
    - `live_search` : Recherche hybride (DataForSEO + SerpAPI + Google Places).
    - `market_diagnosis` : Diagnostic de domaine via 4 appels DataForSEO parallèles.
    - `market_diagnosis_page_targeting_v2` : Matrice de décision (8 axes × 8 verdicts).
    - `escalate_to_phone` : Escalade avec approbation explicite utilisateur.

## 2. Stack & Points de vigilance
- **Fichiers clés** :
    - `supabase/functions/copilot-orchestrator/skills/registry.ts` : Définitions et handlers.
    - `supabase/functions/copilot-orchestrator/personas.ts` : Politiques d'accès (auto vs approval).
- **Stockage & Audit** :
    - Tables : `copilot_actions` (logs exécution), `copilot_sessions` (contextes persistés), `sav_quality_scores` (feedback après escalade).
- **Vigilance** :
    - Respect des quotas par plan utilisateur.
    - Protection contre le "hallucination bypass" sur les données live.
    - Nettoyage des numéros de téléphone (RGPD/Normalisation).

## 3. Architecture des flux
1. **Appel LLM** : L'orchestrateur reçoit une intention nécessitant un outil.
2. **Registry Lookup** : Validation de l'existence du skill et du schéma de paramètres.
3. **Policy Check** : Vérification dans `personas.ts` (ex: `escalate_to_phone` impose `approval`).
4. **Execution Hybrid** :
    - `live_search` : Aggrégation de sources externes avec fallback.
    - `market_diagnosis` : Parallélisation via `Promise.all` pour minimiser la latence.
5. **Post-processing** : Injection des résultats dans le contexte de session (`mem://`).

## 4. Qualité, Coûts & Quotas
La maîtrise des coûts API externes (DataForSEO, SerpAPI) est critique pour la marge brute.

- **Quotas Live Search** :
    - **Free** : 3 appels par conversation.
    - **Premium** : 5 appels par conversation.
    - **Pro** : 20 appels par jour glissant.
- **Quotas Market Diagnosis** :
    - **Free** : 1 par jour.
    - **Premium** : 3 par jour.
    - **Pro** : 10 par jour.
- **Modèle de Coût** :
    - Calcul du coût cumulé par conversation via `llmCostCalculator.ts` + estimation des frais DataForSEO (moyenne 0.05$ / diagnostic complet).

## 5. Benchmark & Références
Comparaison de la pertinence des résultats live et de la profondeur de l'analyse de marché.
- **Perplexity Pro** : Référence pour la synthèse de sources multiples en live search.
- **ChatGPT Search** : Benchmark sur l'intégration de l'UI de recherche.
- **TODO Utilisateur** : Évaluer la latence perçue sur les 4 appels parallèles DFS vs affichage progressif.

## 6. Risques & Mitigations
- **Risque d'épuisement de quota** : Mitigation via `analytics_events` et compteurs dans `copilot_sessions.context`.
- **Incohérence Matrix V2** : Risque de verdict contradictoire entre axes. Mitigation par pondération déterministe (`business_model`).
- **RGPD (Escalade)** : Le numéro de téléphone doit expirer et être masqué dans les logs après 48h (implémenté via `phone_callback_expires_at`).

## 7. Plan d'Action & Requêtes
Audit de l'usage réel sur les 30 derniers jours pour ajuster les garde-fous.

```sql
-- Analyse de l'usage des nouveaux skills
SELECT 
    skill_name, 
    status,
    count(*) as total_executions,
    avg(extract(epoch from (updated_at - created_at))) as avg_latency_sec
FROM copilot_actions
WHERE skill_name IN ('live_search', 'market_diagnosis', 'escalate_to_phone')
GROUP BY skill_name, status
ORDER BY total_executions DESC;
```
