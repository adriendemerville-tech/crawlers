# Audit Brief — Breathing Spiral (v5)
Date : 2026-07-08 · Auditeur : Lovable · Baseline : première itération dédiée algorithme

## 1. Objectif métier
Moteur homéostatique du Cocoon : oscille entre contraction (consolidation Ring 1) et expansion (Ring 2/3) pour maximiser croissance SEO sans cannibalisation ni saturation.

## 2. Cas d'usage principaux
- Autopilot Parménion prescrit une tâche → moteur ajuste priorité selon phase
- Stratège Cocoon booste consolidation (contraction ≥50) ou création (expansion <25)
- Copilot Stratège explique la phase courante à l'utilisateur (skill `explain_spiral_phase`)
- Admin surveille dérive (cron 6h `compute-spiral-signals`)

## 3. Logique métier clé
- 6 signaux pondérés → spiral_score 0-100 (Velocity Decay, Competitor Momentum, Cluster Maturity, GMB Urgency, Conversion Weight, Topic Saturation)
- Phases : Contraction ≥50 (consolidation ×1.3, création ×0.7) / Expansion <25 (création ×1.3, maintenance ×0.9) / Neutre
- Classification rings : `spiralClassifier.ts` (Jaccard token overlap), création `cluster_definitions`
- Garde-fous : Editorial Guard Parménion, Cluster Diversity, Persona Decomposition, Dedup Synonym Layer, Anti-Hallucination Gates
- Reward loop : signal réel post-déploiement réinjecté
- Contraintes mémoire : `breathing-spiral-engine-v5-fr`, `breathing-spiral-strategy-fr`, `breathing-spiral-reward-loop-fr`, `editorial-guard-parmenion-fr`, `cluster-diversity-engine-fr`, `persona-decomposition-engine-fr`, `dedup-synonym-layer-fr`, `anti-hallucination-gates-fr`, `anti-cannibalization-logic-fr`

## 4. Critères de qualité attendus
| Axe | Cible | Mesure |
|---|---|---|
| Cohérence Copilot | skill `explain_spiral_phase` + vector memory sur décisions | `copilot_actions` |
| Fonctionnel | phase computed < 26h pour tout site actif | `select max(updated_at) from spiral_signals` |
| Anti-saturation | 0 création dans cluster ≥10 articles (garde -15pts + Layer D blocker HTTP 409) | logs `iktracker-actions` |
| Anti-cannibalisation | 0 duplicate d'intent Jaccard >0.6 sur 30j | audit `cluster_definitions` |
| Performance | cron `compute-spiral-signals` < 5 min tout site | logs edge |
| Coût LLM | < 4¢ / cycle Parménion | `ai_gateway_usage` |
| Traçabilité | chaque décision loggée avec spiral_score + phase + signaux | table `parmenion_decisions` |

## 5. Benchmark concurrentiel
- **MarketMuse Cluster Planner** — scoring topique, pas homéostatique
- **Frase Topic Graph** — visualisation, faible sur décision auto
- **Surfer Grow Flow** — recommande consolidation, monolithique
- **NeuronWriter Semantic Optimizer** — bon sémantique, pas de boucle reward
- Barre : rester le seul moteur homéostatique avec 6 signaux + reward loop + phase oscillante (breathing = signature Crawlers)

## 6. Périmètre d'audit
### In scope
- Backend : `compute-spiral-signals`, `_shared/spiralClassifier.ts`, `cocoon-strategist`, `iktracker-actions` (Layer D), Editorial Guard, Persona Decomposition, Dedup Synonym Layer
- Frontend : cartes phase spirale dans Cocoon, indicateurs Parménion
- Tables : `spiral_signals`, `cluster_definitions`, `keyword_universe`, `parmenion_decisions`
### Hors scope
- Marina outreach (#6)
- Content Architect (#3, séparé)

## 7. Signaux à mesurer
- SQL phase distribution : `select phase, count(*) from spiral_signals where updated_at > now()-interval '7 days' group by phase`
- Saturation guard hits : logs `iktracker-actions` filter "Layer D"
- Coût : `select avg(total_cost) from ai_gateway_usage where edge_function in ('compute-spiral-signals','cocoon-strategist')`
- Reward loop : delta position/traffic 30j post-déploiement vs prédiction
- Marina : vérifier que TOUS les modules Marina utilisent les signaux spirale (cross-check avec brief #6)
